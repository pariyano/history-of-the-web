Implementation report

This page explains how the website was planned, structured and styled.

Structure choice

We did not use a pre-made template for this website. Instead, we created a simple custom static website using only HTML and CSS. This choice helped us better understand how a website is structured and gave us direct control over the pages, navigation, layout, colors and fonts.

Information architecture

The menu has four pages. The Home page introduces the topic. The Timeline page explains the historical development of the Web. The How It Works page explains the technical process behind opening a website. The Report page documents our design and implementation choices. This order moves from general context to technical explanation and then to reflection on our own work.

Stylistic choices

We used nude and neutral colors because the topic is historical and educational. The soft background makes the website calm and readable. The rounded cards and simple typography create a modern but minimal style. In CSS, we changed properties such as background-color, color, font-family, padding, margin, border-radius and box-shadow.

HTML and CSS logic

The HTML structure uses semantic tags. The <header> contains the title and navigation. The <main> contains the central content. The <section> elements divide the page into logical blocks. The <footer> gives project information. The external CSS file, styles.css, is connected to all pages with the <link> tag, so the design remains consistent across the whole site.

Challenges

1. Learning and understanding the role of each CSS element used in the website. Since we did not use a pre-made template, we had to understand how every CSS property affected the final visual result, such as colors, spacing, margins, padding, borders, font size, layout, and responsive design. 2. Creating a navigation system that was clear but not too complex. 3. Making the design work well on smaller screens, such as phones or tablets.
